# NotesSystem



NotesSystem — консольная система заметок, мониторинга и администрирования для ИТ-инфраструктуры малого VPN-сервиса.

Проект разработан в рамках курсовой работы по разработке, тестированию и отладке программного обеспечения.

## Назначение системы

Система предназначена для работы в ИТ-ландшафте малого бизнеса, предоставляющего VPN-сервис.

С помощью консольного клиента администратор или пользователь может создавать заметки на серверах, просматривать логи безопасности, получать статистику загрузки устройств, а также проверять наличие обновлений приложения через GitHub.

## Основные возможности

\- авторизация пользователей;

\- разделение прав доступа по ролям;

\- создание и просмотр заметок;

\- редактирование, удаление и восстановление заметок;

\- просмотр логов безопасности;

\- просмотр локальной статистики CPU/RAM/HDD;

\- сбор статистики агентом NotesWatcher;

\- сохранение статистики в PostgreSQL;

\- проверка обновлений через GitHub;

\- отдельный установщик NotesInstaller.

## Компоненты системы

### NotesCli

Основной консольный клиент.

Используется для:

\- авторизации пользователей;

\- работы с заметками;

\- просмотра логов безопасности;

\- просмотра статистики устройств;

\- проверки обновлений через GitHub.

### NotesInstaller

Учебный установщик приложения.

Выполняет следующие действия:

\- создаёт рабочую папку приложения;

\- создаёт папку для логов;

\- создаёт папку для конфигурации;

\- формирует файл `config.json`;

\- проверяет подключение к PostgreSQL.

### NotesWatcher

Агент мониторинга.

Работает как отдельное консольное приложение и выполняет следующие действия:

\- получает имя текущего устройства;

\- получает IP-адрес;

\- собирает статистику CPU;

\- собирает статистику RAM;

\- собирает статистику HDD;

\- сохраняет статистику в PostgreSQL.

### NotesShared

Общая библиотека классов.

Содержит:

\- модели данных;

\- сервисы бизнес-логики;

\- подключение к базе данных;

\- хеширование паролей;

\- работу с заметками;

\- работу с логами безопасности;

\- работу со статистикой;

\- проверку обновлений.

## Роли пользователей

В системе используются три роли приложения.

| Роль | Описание |

|---|---|

| `admin` | Администратор. Имеет доступ ко всем функциям системы |

| `user` | Пользователь. Может работать со своими заметками |

| `statistic` | Пользователь статистики. Может просматривать статистику CPU/RAM/HDD |

## Тестовые пользователи

| Логин | Пароль | Роль |

|---|---|---|

| `admin` | `admin123` | `admin` |

| `user1` | `admin123` | `user` |

| `stat` | `admin123` | `statistic` |

## Команды NotesCli

### Общие команды

```bash

\--help

\--version

exit

```

### Авторизация

```bash

\--login <username> <password>

\--logout

\--myrole

```

Примеры:

```bash

\--login admin admin123

\--login user1 admin123

\--login stat admin123

```

### Заметки

```bash

\--addNewNote "Текст заметки"

\--listNotes

\--editNote <id> "Новый текст"

\--deleteNote <id>

\--restoreNote <id>

```

Примеры:

```bash

\--addNewNote "Проверить сервер БД"

\--listNotes

\--editNote 1 "Проверить сервер БД после обновления"

\--deleteNote 1

\--restoreNote 1

```

### Логи безопасности

```bash

\--securityLogs list

```

Команда доступна только пользователю с ролью `admin`.

### Статистика

```bash

\--systemStats local

\--systemStats history

```

Команда `--systemStats local` показывает текущую загрузку локального устройства.

Команда `--systemStats history` показывает последние записи статистики, сохранённые агентом NotesWatcher в базе данных.

Команды статистики доступны ролям:

```text

admin

statistic

```

### Обновления

```bash

\--checkUpdate

\--update

```

Команда `--checkUpdate` получает файл `version.json` из GitHub-репозитория и сравнивает локальную версию приложения с актуальной версией.

Команда `--update` скачивает архив новой версии, если обновление доступно.

## Автообновление

Для проверки обновлений используется файл:

```text

releases/version.json

```

Пример содержимого файла:

```json

{

&#x20; "version": "1.0.0",

&#x20; "downloadUrl": "https://github.com/Vitalya1357/NotesSystem/releases/download/v1.0.0/NotesSystem.zip"

}

```

Приложение получает этот файл из GitHub и сравнивает версию из него с локальной версией приложения.

Локальная версия хранится в классе `AppConfig`.
