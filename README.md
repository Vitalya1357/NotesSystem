# NotesSystem

NotesSystem — это консольное приложение для работы с заметками, просмотра системной статистики и журнала безопасности.

Проект написан на C# с использованием .NET Framework 4.7.2.  
В качестве базы данных используется PostgreSQL.

## Назначение проекта

Приложение предназначено для хранения служебных заметок и просмотра информации о состоянии устройства.

В системе реализованы:

- регистрация пользователей;
- вход по логину и паролю;
- выход из учетной записи;
- хранение паролей в виде хеша;
- работа с заметками;
- разграничение прав по ролям;
- просмотр журнала безопасности;
- просмотр статистики CPU, RAM и HDD;
- проверка и скачивание обновлений через GitHub.

---

Для запуска проекта нужны:

- Windows 10 или Windows 11;
- Visual Studio;
- .NET Framework 4.7.2;
- PostgreSQL;
- pgAdmin;
- Git;
- GitHub.

## Основные проекты

Решение состоит из нескольких проектов.

`NotesCli` — основной консольный клиент.  
В нем пользователь вводит команды, входит в систему, работает с заметками, смотрит статистику и проверяет обновления.

`NotesInstaller` — установщик приложения.  
Он создает нужные папки, конфигурационные файлы и проверяет подключение к базе данных.

`NotesWatcher` — агент мониторинга.  
Он собирает данные о компьютере: CPU, RAM, HDD, имя устройства и IP-адрес.

`NotesShared` — общая библиотека.  
В ней находятся модели, сервисы, подключение к базе данных, хеширование паролей и общая логика.

`NotesUnitTests` — проект с unit-тестами.  
Он используется для проверки авторизации, строк подключения, заметок и обновлений.

## База данных

В проекте используется база данных:

```text
NotesSystemDB
```

Основные таблицы:

```text
roles
users
notes
devices
system_metrics
security_logs
app_versions
db_connection_strings
```

---

## Роли

В таблице `roles` используются три роли:

```text
admin
user
statistic
```

Пример добавления ролей:

```sql
INSERT INTO roles(name)
VALUES
('admin'),
('user'),
('statistic');
```

---

## Пользователи

В таблице `users` хранятся пользователи приложения.

Основные поля:

```text
id
username
password_hash
role_id
failed_login_attempts
created_at
```

Пароли не хранятся в открытом виде.  
Перед сохранением пароль хешируется.

Пример хеша для пароля `admin123`:

```text
240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9
```

Флаг `is_blocked` в текущей версии не используется.

Если администратор удаляет пользователя, то строка пользователя полностью удаляется из таблицы `users`.

## Строки подключения

Для каждой роли используется отдельная строка подключения к PostgreSQL.

Они хранятся в таблице:

```text
db_connection_strings
```

Пример:

| Роль | Пользователь PostgreSQL |
|---|---|
| admin | notes_admin |
| user | notes_user |
| statistic | notes_statistic |

После входа приложение определяет роль пользователя и использует строку подключения, которая соответствует этой роли.

Например:

```text
user1 входит в приложение
приложение определяет роль user
после этого используется подключение notes_user
```

Это нужно для того, чтобы права проверялись не только в коде, но и на уровне базы данных.

---

## Права доступа

В системе есть три основные роли.

`admin` — администратор.  
Он может управлять пользователями, смотреть журнал безопасности, работать с заметками и смотреть мониторинг.

`user` — обычный пользователь.  
Он может работать только со своими заметками.

`statistic` — пользователь для просмотра статистики.  
Он может смотреть мониторинг, но не может управлять пользователями и работать с заметками.

---

## Таблица прав

| Возможность | admin | user | statistic |
|---|:---:|:---:|:---:|
| Вход в систему | Да | Да | Да |
| Выход из системы | Да | Да | Да |
| Регистрация | Да | Да | Да |
| Просмотр своей роли | Да | Да | Да |
| Создание заметок | Да | Да | Нет |
| Просмотр заметок | Да | Только свои | Нет |
| Редактирование заметок | Да | Только свои | Нет |
| Удаление заметок | Да | Только свои | Нет |
| Управление пользователями | Да | Нет | Нет |
| Просмотр журнала безопасности | Да | Нет | Нет |
| Просмотр мониторинга | Да | Нет | Да |
| Проверка обновлений | Да | Да | Да |

Если у пользователя нет прав, приложение выводит понятное сообщение.

Примеры сообщений:

```text
Нет прав для управления пользователями.
Нет прав для просмотра журнала безопасности.
Нет прав для просмотра мониторинга.
Нет прав для работы с заметками.
```

## Обновления через GitHub

Для обновления используется GitHub Release.

Порядок выпуска новой версии:

1. Изменить версию приложения в коде.
2. Собрать проект в режиме Release.
3. Создать архив:

```text
NotesSystem.zip
```

4. Создать новый Release на GitHub.
5. Прикрепить архив `NotesSystem.zip`.
6. Обновить таблицу `app_versions`.

Пример ссылки на архив:

```text
https://github.com/Vitalya1357/NotesSystem/releases/download/v1.0.1/NotesSystem.zip
```

Пример обновления записи в базе данных:

```sql
UPDATE app_versions
SET
    version = '1.0.1',
    download_url = 'https://github.com/Vitalya1357/NotesSystem/releases/download/v1.0.1/NotesSystem.zip',
    created_at = NOW()
WHERE id = 1;
```

После этого старая версия приложения сможет найти обновление через команду:

```bash
--checkUpdate
```

И скачать его через команду:

```bash
--update
```

## XML-файл с тестовыми данными

Для тестов используется файл:

```text
NotesUnitTests\TestData\test-input-data.xml
```

Он нужен для того, чтобы один тест запускался несколько раз с разными входными данными.

Например, один тест авторизации проверяет:

- успешный вход;
- неверный пароль;
- пустой логин;
- пустой пароль.

Для этого используется:

```csharp
[DynamicData]
```

## Что используется в тестах

В тестах используются:

```text
[TestClass]
[TestMethod]
[DataTestMethod]
[DynamicData]

```

Проверки строк подключения выполняются без реального подключения к PostgreSQL.
